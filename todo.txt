Github gebruikt een credential manager om via tokens met github te praten.
Je moet read / write naar deze credential manager. Per gebruiker kan je dan in
een .env file de gerelateerde key:value pairs opslaan in de vorm van USER:ACCESS_TOKEN.

1. zoek git link info op in config files (git_shell.js)
2. verzamel gebruikersinfo via prompt en check of deze bestaan
3. verdeel link via url_parser
4. check voor link aanwezigheid in wincred_manager
  --> aanwezig = veranderen met nieuwe username
  --> niet aanwezig = 2 nieuwe entries voor git repository

(optioneel: check of gebruiker toegang heeft tot repository)

5. git config --global user.name / user.email veranderen naar values in access.json file

TIPS:

package.json --> "bin" om script te specificeren en daarna globaal installeren via npm
